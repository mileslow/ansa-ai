# Valuable booklet and instruction source additions

Date acquired and verified: 2026-07-17

## Outcome

Eight high-value public source files were added to the current phase 04, 06, and 07 directories. The additions close the reusable component-template gap, create a genuine consecutive-year same-employer guide pair, add two unambiguously broker-authored checklists plus one general-agency implementation checklist, add current official legal/model material, and add a real plan-change/override example.

No completed employer decision record was scraped or fabricated. That role still requires controlled synthetic fixtures or properly authorized client records.

## Same-employer pair verification

The Seminole County files are a real annual pair, not two renamed copies:

- Both are official Seminole County Employee Benefits Guides and both list Margaret Gee in PDF metadata.
- The 2025 file covers 2025, has 49 pages, and has SHA-256 `e8a6b5a43fcc24e6b363c941fe3c90d3be26af58fb59256925d2ba234ce4930c`.
- The existing 2026 file covers 2026, has 56 pages, and has SHA-256 `a345fc5dc2d91feae180e9aa18f35b385b84e6f30b498fbcd4d3beebd61fd6a9`.
- Extracted tables of contents and annual pages show substantive differences, including page count, account limits, voluntary-benefit organization, glossary placement, and annual-notice placement.

## Verification performed

For every added PDF:

- Confirmed a valid PDF signature and successful `pdfinfo` parse.
- Confirmed the file is not encrypted and recorded page counts.
- Extracted text with `pdftotext` and checked publisher, title, effective year, and claimed source role.
- Rendered page 1 with Poppler and visually inspected it for corruption, clipping, wrong-document downloads, and obvious sensitive records.

For both DOCX files:

- Confirmed valid OOXML packages with `unzip -t`.
- Extracted and reviewed document text and core metadata.
- Generated and visually inspected macOS Quick Look first-page previews.
- A full LibreOffice multi-page render was not available because LibreOffice/`soffice` is not installed in this workspace. Structural verification passed, but this is not equivalent to the document skill's full render gate.

Privacy/safety review found no filled employee census, claim record, Social Security number, Tax ID, bank data, or participant-level personal record. The completed employer guide contains publicly posted organizational/vendor contacts, and some files retain public author metadata. These are acceptable for provenance but must not be copied into another employer's output.

## Rejected candidates during QA

- A public Columbia University 2026 SMM was rejected and removed because the posted PDF was labeled `Draft V1`.
- A Risk Strategies/eBen 2025 checklist was rejected and removed because every page was marked `Proprietary & Confidential`.
- The current DOL Marketplace and COBRA download URLs were checked against the retained files and were byte-identical. Re-downloading them would not cure the stale 2023/2024 paragraphs or expired OMB date already identified in the audit, so no duplicate was added.
- No public full benefits-booklet template with a clear permissive license was found. The DOL SBC DOCX satisfies the requested reusable **component-template** role with a federal public-domain basis.

## Remaining controlled-fixture needs

The repository still lacks safe completed examples of:

- Employer plan selections and contribution decisions.
- Broker/client override emails or exported messages.
- Final employer approval/sign-off records.
- A reusable full-booklet template whose license explicitly permits adaptation.

Do not source real completed employer decision forms from public search. Create controlled synthetic fixtures with invented employer names, contacts, group numbers, rates, dates, elections, and signatures, clearly labeled `SYNTHETIC / NON-PRODUCTION`, or use client records only under an explicit data-use authorization and redaction policy.
